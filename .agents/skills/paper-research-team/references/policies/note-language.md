# Note Language Policy

Use this when writing or updating reusable Obsidian paper notes.

Default from `config.yaml`:

- `obsidian.language: "ko"`
- `obsidian.note_body_language: "ko"`

## Rule

Write explanatory section body content in Korean.

Preserve original wording for:

- paper title
- author names
- venue names
- DOI, URL, Zotero key, filenames, and paths
- frontmatter keys and tag names
- template headings unless the template itself changes
- technical terms and named concepts, such as `takeover request`, `open path mode`, `sensory substitution`, `stimulus-response compatibility`

## Style

Prefer Korean explanation with original technical terms embedded inline.

Example:

`Open path mode는 traversable area를 직접 안내하는 action-oriented feedback으로 볼 수 있다.`

Do not translate a term when translation would make later search, citation, or concept matching harder.

## Existing Notes

Do not rewrite an existing note solely to change language unless the user asks. When updating a note for substantive reasons, apply this policy to the changed body sections.
