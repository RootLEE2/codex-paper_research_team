# Blocked Output Schemas

Use these only when required selected papers still lack PDFs, preprints, or confirmed full text.

## Blocked Synthesis

```markdown
# Synthesis: <topic>

status: blocked
reason: Required selected papers still lack PDFs, preprints, or confirmed full text.

## Pending Inputs

Link to `pending-papers.md` and relevant records under `papers/obsidian/pending-notes/`.

## Resume Condition

Resume after the missing full text is available, or after the user explicitly narrows/defers those papers.
```

## Professor Review

For normal unblocked review:

```markdown
# Professor Review: <topic>

## Decision

Blocked, pass, or fail.

## Review Mode

Blocked, non-final-partial, or final.

## Major Gaps

## Missing Seminal Work

## Missing Recent Work

## Weak Claims

## Pending Full-Text Risk

## Required Next Cycle

## Optional Next Cycle
```

For blocked review placeholders:

```markdown
# Professor Review: <topic>

status: blocked
reason: Final professor review requires unblocked synthesis from full-text analyzed papers.

## Pending Inputs

## Resume Condition
```
